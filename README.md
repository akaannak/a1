# Немного про данные:
Структура CSV следующая (столбцы) :
N	- количество случайно сгенерированных точек (число испытаний)
wide_area - приближённая оценка площади фигуры при использовании «широкой» прямоугольной области генерации точек
tight_area - приближённая оценка площади фигуры при использовании «узкой» прямоугольной области генерации точек
wide_error - относительная ошибка результата для широкой области по сравнению с точным значением
tight_error	- относительная ошибка результата для узкой области

# График первого типа
<img width="2400" height="1200" alt="plot_area" src="https://github.com/user-attachments/assets/4d7480bd-21a9-42ab-ab78-b595553ad1c5" />
Отображает, как меняется приближенное значение площади в зависимости от указанных параметров алгоритма.

Широкая область:
- в начале наблюдаются большие колебания: оценки скачут
- даже после увеличения числа точек до 100000 оценка медленно сходится к истинному значению площади
— метод шумный

Узкая область
- уже при N ≈ 200–500 оценка становится стабильной
- линия быстро выходит к истинному значению и практически не колеблется вокруг него
- при N ≥ 5000 S(N) почти идеально совпадает с точным значением

Вывод:
Узкая область обеспечивает быструю сходимость и значительно меньшие колебания, а широкая — медленную сходимость и большие отклонения из-за высокого уровня шума. Увеличение числа испытаний приводит к уменьшению дисперсии оценки (о чем и говорит нам метод Монте-Карло).

# График второго типа
<img width="2400" height="1200" alt="plot_error" src="https://github.com/user-attachments/assets/7d88d5a3-930f-45b0-8918-a1d90522fd02" />
Отображает, как меняется величина относительного отклонения приближенного значения площади от ее точной оценки в зависимости от указанных параметров алгоритма.

График ошибки подтверждает наблюдения выше.
Широкая область:
- ошибка в начале может достигать 25%
- даже при N = 20000 ошибка остаётся в районе 2–3%
- при N = 100000 ошибка медленно снижается до примерно 0.5–1%

Узкая область:
- ошибка уже при N ≈ 1000 падает до менее 1%
- после N > 10000 стабильно держится на уровне 0–0.3%
- ошибка практически исчезает при больших N

Вывод:
Узкая область превосходит широкую по точности во всём диапазоне значений N, снижение ошибки происходит на порядок быстрее.

## Общий вывод:
Узкая область моделирования значительно улучшает работу метода Монте-Карло: уменьшает дисперсию, ускоряет сходимость, обеспечивает стабильные оценки площади даже при малом количестве точек. Широкая область даёт существенно большую погрешность, так как точки распределяются на гораздо большей площади, и лишь малая их часть попадает в нужную фигуру.
